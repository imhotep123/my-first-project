## Step 3: ヒアリングシートの読み込みと改正指示の把握

ヒアリングシートの改正指示一覧（B列〜F列）を全行読み込む。

### D列（改正種別）の定義

| 改正種別 | 意味 | 対応方針 |
|----------|------|---------|
| ①新設 | 旧規約にない条文を新設 | 新設条文テンプレートまたはR7から内容を取得し追加 |
| ②全面書き換え | 旧規約を捨ててR7に置き換え | 旧規約の条文を破棄し、R7の対応条文で完全置換 |
| ③不足部分追加 | 旧規約に不足している項・号を追記 | 旧規約の既存内容を維持しつつ、不足部分のみR7から取り込む |
| ④現状のまま | 修正なし | 旧規約をそのまま維持 |

### E列（個別指示）の読み取りポイント

- **条文番号**：E列の「第X条の2として新設」等の記述が新しい条文番号を決定する
- **旧規約の条文番号は原則維持**（リナンバリングは行わない）
- **参照先条文番号の修正**：R7の条文番号を新規約の条文番号に置き換える（後述のマッピング表参照）
- **旧規約固有の値を維持**：管理組合名、事務所所在地、口座情報、遅延損害金率、徴収日等

## Step 5: 改正案.docx の作成・追記

### python-docx のヘルパー関数（標準パターン）

```python
from docx import Document

doc = Document(r'C:/path/to/改正案.docx')  # 既存ファイルの場合
# doc = Document()  # 新規作成の場合

def add_chapter(text):
    """第X章 見出し（中央揃え・太字）"""
    p = doc.add_paragraph()
    p.alignment = 1  # CENTER
    run = p.add_run(text)
    run.bold = True

def add_section(text):
    """第X節 見出し（中央揃え・太字）"""
    p = doc.add_paragraph()
    p.alignment = 1
    run = p.add_run(text)
    run.bold = True

def add_title(text):
    """（条文名）見出し（左揃え）"""
    p = doc.add_paragraph()
    p.add_run(text)

def add_body(text):
    """条文本文（左揃え）"""
    p = doc.add_paragraph()
    p.add_run(text)

doc.save(r'C:/path/to/改正案.docx')
```

### 条文テキストの書式ルール

- **章見出し**: `第６章\u3000管理組合`（全角スペース `\u3000` で区切る）
- **節見出し**: `第１節\u3000組織`（同上）
- **条文名**: `（届出義務）`（丸括弧は全角）
- **条文本文**: `第３３条\u3000新たに組合員の資格を取得し…`（条番号と本文の間は `\u3000`）
- **項番号**: `２\u3000組合員は…`（半角算用数字＋全角スペース）
- **号番号**: `一\u3000管理組合が管理する…`（漢数字一字＋全角スペース）
- **イロハ番号**: `イ\u3000敷地及び…`（同上）

### 既存ファイルへの追記時の重複防止

改正案.docxが既存の場合は必ず以下を実施する：

1. 現在の段落数と末尾内容を確認（`print(len(doc.paragraphs))`, `print(doc.paragraphs[-1].text)`）
2. どの条文まで記載済みか確認
3. 次に追加する条文から開始し、既存部分には触れない
4. **「又は電子署名をもって設定され発効する」で終わる迷子段落**がある場合は最初に削除する：
   ```python
   for i in range(len(doc.paragraphs)-1, -1, -1):
       if '又は電子署名をもって設定され発効する' in doc.paragraphs[i].text:
           doc.paragraphs[i]._element.getparent().remove(doc.paragraphs[i]._element)
           break
   ```

### 重複コンテンツの修正（緊急時）

誤って同じ内容が複数回追加された場合は以下で修正する：

```python
# 正しい末尾インデックスを特定してそれ以降を全削除
cutoff_text = '４　修繕積立金については、管理費とは区分して経理しなければならない。'
cutoff_idx = None
for i, p in enumerate(doc.paragraphs):
    if cutoff_text in p.text:
        cutoff_idx = i
# インデックス (cutoff_idx + 1) 以降をすべて削除してから正しい内容を1回だけ追加
while len(doc.paragraphs) > cutoff_idx + 1:
    p = doc.paragraphs[-1]._element
    p.getparent().remove(p)
```

## Step 6: 改正種別ごとの処理ガイドライン

### ①新設

- E列の「第X条の2として新設すること」等の指示に従い条文番号を決定
- 新設条文テンプレート（`新設する条文（種別）.docx`）またはR7から条文内容を取得
- R7内の参照条文番号をマッピング表で変換してから記載
- 位置はB列の旧規約条文番号の直後に挿入

### ②全面書き換え

- R7の対応条文（C列に記載）のテキストを取得
- 旧規約の条文番号（B列）を維持し、条文名・本文をR7で完全置換
- 参照条文番号はすべてマッピング表で変換
- 例外として以下は旧規約固有の値を維持：
  - 管理組合名・マンション名（第6条）
  - 事務所所在地（第6条）
  - 口座情報（第62条相当）
  - 遅延損害金率（第60条相当）
  - 徴収日（第60条相当）

### ③不足部分追加

- 旧規約の既存条文内容をベースに維持
- R7に存在してE列で指摘された不足項・号のみを追加
- 追加位置は原則として旧規約の最終項の後
- 参照条文番号はマッピング表で変換

### ④現状のまま

- 旧規約の条文をそのままコピー
- 参照条文番号の修正も不要

## Step 7: 完了検証

スクリプト実行後、以下を確認してユーザーに報告する。

```python
import sys, re
sys.stdout.reconfigure(encoding='utf-8')
from docx import Document
doc = Document(r'C:/path/to/改正案.docx')
texts = [p.text for p in doc.paragraphs]

# 1. 総段落数
print(f'総段落数: {len(texts)}')

# 2. 章・節・付則の構造
for i, t in enumerate(texts):
    if any(k in t for k in ['第１章','第２章','第３章','第４章','第５章','第６章','第７章','第８章','付則']):
        print(f'[{i}] {t}')
    elif t.startswith('第') and '節' in t:
        print(f'  [{i}] {t}')

# 3. 重複チェック（完全形の条文番号で）
article_pat = re.compile(r'^第[０-９\d]+条(の[０-９\d]+)?[\u3000\s]')
seen = {}
for i, t in enumerate(texts):
    m = article_pat.match(t)
    if m:
        key = m.group(0).rstrip()
        seen.setdefault(key, []).append(i)
dups = {k:v for k,v in seen.items() if len(v)>1}
# 付則の第1条は本文の第1条（目的）と重複するため正常
if dups:
    for k, v in dups.items():
        print(f'重複: {k} @ {v}')
else:
    print('重複なし')
```

報告内容：
- ファイルパス
- 総段落数
- 章・節・付則の構造リスト
- 重複の有無
- ヒアリングシートの全項目に対応できたかの確認

## 注意事項

- **コンテキスト不足時の継続**: セッション終了後も継続できるよう、どの条文まで記載したかをユーザーに報告してから終了する
- **R7テキストは長大**なため Grep を積極活用。`第X条` で検索して前後の範囲を Read で取得する
- **付則の第1条は本文の第1条と重複しても正常**（日本の規約の標準形式）
- **団地型・複合用途型**では棟管理組合・団地管理組合の二層構造に注意。条文番号体系が単棟型と異なる
- **複合用途型**では住宅部分と非住宅部分の費用負担割合・使用制限の条文が追加される
- **電磁的方法あり版を標準として使用**。電磁的方法なし版を使う場合はユーザーに確認する
- 旧規約に固有の独自条項（民泊禁止の独自方式、遅延損害金率、口座情報等）は**必ず維持**する
- Write ツールで新規ファイルを書く場合は先に Read ツールで対象ファイルを読む必要がある（制約回避のため Bash で python スクリプトを書いて実行する方式を使う）
